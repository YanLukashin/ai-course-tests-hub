# Межмодульные тесты курса ИИ

Статическая веб-платформа для межмодульного тестирования по модулям 1–6.

Что умеет текущая версия:
- отдельный тест для каждого модуля;
- автопроверка закрытых вопросов;
- сохранение прогресса и результатов в браузере студента;
- готовность к статическому хостингу без backend.

## Запуск локально

1. Собрать данные из markdown-файлов тестов:

```bash
node scripts/build-course-data.mjs
```

2. Поднять локальный сервер:

```bash
python3 -m http.server 4173
```

3. Открыть в браузере:

`http://localhost:4173`

## Откуда берутся данные

Сборщик читает тесты из родительского проекта:

- `../Модуль 1/Module-1-Test-Final.md`
- `../Модуль 2/Test-Module-2.md`
- `../Модуль 3/Module-3-Test-Ready.md`
- `../Модуль 4/Module-4-Test-Final.md`
- `../Модуль 5/Test-Module-5-NoCode-Automation.md`
- `../Модуль 6/Module-6-Test-Ready.md`

На выходе формируется:

- `data/course-data.json`

## Как обновлять тесты

1. Правите markdown-файл нужного модуля.
2. Запускаете:

```bash
node scripts/build-course-data.mjs
```

3. Публикуете обновлённую папку платформы.

## Структура

- `index.html` — входная точка.
- `app/main.js` — интерфейс хаба и логика сдачи тестов.
- `app/styles.css` — стили платформы.
- `scripts/build-course-data.mjs` — сборка JSON из тестовых markdown-файлов.
- `data/course-data.json` — готовые данные для фронтенда.

## Размещение

Платформа статическая, поэтому её удобно размещать на:

- GitHub Pages — если нужен простой публичный деплой из репозитория;
- Cloudflare Pages — если нужен быстрый CDN-деплой без сервера;
- Netlify — если нужен такой же статический хостинг с простым интерфейсом.

Важно: в текущей архитектуре результаты хранятся только в браузере пользователя. Если понадобится централизованный сбор результатов, следующим шагом лучше добавить внешний приёмник результатов: Google Sheets через Apps Script, Supabase или Firebase.

## Публикация на GitHub Pages

В проект уже добавлен workflow:

- `.github/workflows/deploy-tests-pages.yml`

Как он работает:

1. Берёт код из ветки `main`.
2. Пересобирает `data/course-data.json` из markdown-тестов модулей 1–6.
3. Публикует содержимое папки `Платформа курса ИИ` в GitHub Pages.

Что нужно сделать на GitHub один раз:

1. Залить весь проект в репозиторий, включая папки `Модуль 1` ... `Модуль 6` и `Платформа курса ИИ`.
2. Открыть `Settings -> Pages`.
3. В разделе `Build and deployment` выбрать `Source: GitHub Actions`.
4. Убедиться, что основная ветка называется `main`.

После этого публикация будет происходить автоматически при изменениях в:

- `Платформа курса ИИ/**`
- markdown-тестах модулей 1–6

Если вы меняете только тексты тестов, вручную пересобирать JSON перед пушем не обязательно: workflow сделает это сам на GitHub.

## Чек перед деплоем

Перед публикацией локально достаточно проверить:

1. Сборку данных:

```bash
node scripts/build-course-data.mjs
```

2. Локальный просмотр:

```bash
python3 -m http.server 4173
```

3. Что на главной странице открываются все модули, а вопросы и ответы отображаются корректно.
